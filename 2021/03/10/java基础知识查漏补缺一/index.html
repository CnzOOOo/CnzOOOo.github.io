    java开发环境

    1. java开发环境的搭建

      1. 下载安装jdk

      2. 配置环境变量

        1）JAVA_HOME:jdk的安装路径

        2）path:jdk下bin文件夹的路径（建议在path值的最前面添加jdk下的bin的目录即可）

        3）classpath:值的最前边加**”.”**

    2. jdk、jre、jvm的关系与区别
      1. jvm:java虚拟机
      2. jre:java运行环境，jre=jvm+java系统类库
      3. jdk:java开发环境，jre+开发工具包（编译、运行工具等）
    3. java程序的运行机制

      1. 先编译后运行

        1）先编译：.java源文件编译成.class字节码文件

        2）后运行：加载并执行.class字节码文件

    4. Eclipse中如何开发应用程序

      1. eclipse中写好程序后保存，.java源文件就已经编译好了，产生了对应的.class文件

      2. 创建项目：所有字母小写

        ​ 1）包：域名反写.功能 baidu.com.util —- 通用的命名规则

        ​ 2）类：首字母大写，如果类名由多个单词组成，每个单词的首字母大写

    变量

    1. 变量的声明

      1. 数据类型+变量名； 如：int a;

      2. 变量的命名规则：5点

        ​ 1）由字母、数字，_，$ 组成，数字不能开头

        ​ 2）严格区分大小写

        ​ 3）不可以使用java关键字/保留字

        ​ 保留字：java中现在没有使用但是将来可能会用的一些英文单词，java占用了（保留）

        ​ const：常量

        ​ goto: 跳转

        ​ 字面量：null，true，false

        ​ 4）遵循驼峰命名法，见名知意

        ​ 5）中文可以使用，但是不建议使用

      3. 若多个变量的数据类型相同，此时可以这样声明：

        int a,b,c=0,d=9; —-正确的

    2. 变量的初始化：第一次赋值

      1. 在声明的同时初始化

      2. 先声明，后初始化 —-一个变量若没有初始化直接使用，编译错误

    3. 变量的访问

      1. 给变量赋值 —-存值

      2. 取值 —-取变量中的那个数

        成员变量和局部变量：

        ​ 成员变量：定义在类的里边，方法的外边 —- 在类中任何位置都可以使用

        ​ 局部变量：定义在方法的内部 —- 只能在本方法使用

    浮点类型

    1. float:

      float f = 3.4F/f;

    2. double:双精度浮点数

      double d = 3.4;

      Java中出现的小数默认都是double类型，Java中出现的整数默认都是int类型

    问题：单精度和双精度指的是什么？

    ​ 精度：小数的有效位数（不是指小数点后的数字位，而是指从整数部分开始的有效数字）

    ​ eg: 3.1415926 —- 8位有效数字

    ​ float: 单精度 –7/8位有效数字

    ​ double:双精度 –15/16位有效数字

    ​ 精度和编译器有关系，不同的编译器精度有可能不相同

    ​ 计算机底层保存浮点数的时候，保存的是最接近该数的一个小数，不一定四舍五入

    ​ 记住： float double 有时候不能精确的表示浮点数

    ​ 在运算中可能会出现误差，所以double不用于做精确运算，如果要做精确运算，用BigDecimal

    unicode字符集

    ​ 版本3： 65536个整数 一个整数对应世界上的一个字符

    ​ 版本8： 10万+整数

    问题： int 类型的数字用字符表示，也在unicode中对应的整数

    字符集发展：

    ASCII码也是字符集：是编程语言最开始时候的字符集 -只能表示英文，数字，符号 1字节表示一个字符

    各国针对自己的国情：产生了各国的字符集：

    ​ eg: 中国：GB2312字符集：能表示中文，英文，数字，符号 –2字节表示一个字符

    ​ 日本也在ASCII码基础上进行了扩展，2字节表示一个字符，其他语言同理

    基于上述情形，在一个文本中若存在多国语言，那么计算机保存时的整数就产生了冲突，即出现了乱码，

    为了解决这个问题，产生了unicode字符集（国际码，万国码）

    unicode字符集中有10万+整数，每个整数表示世界上一个字符，所有国家的字符都有表示的整数

    ​ 用2个字节表示一个字符

    ​ 是定长字符集 –永远都是用2个字节表示一个字符

    数据类型转换问题

    自动转换（隐式转换）：小转大

    ​ - byte short int long float double

    ​ - char –>int

    ​ - boolean和其他的数据类型都不兼容

    ​ 问题1：char可不可以自动转换成byte –不可以

    强制转换（显示转换）：大转小

    ​ 顺序和以上顺序正好相反，若从大的类型转换为小的类型，需强制转换（）来实现

    ​ 问题2：long类型的数字如果大于8位，转换为float后，后面的丢掉吗？

    ​ float在计算机中保存时都是以科学计数法保存的

    不同类型间运算问题

    ​ 不同数据类型间进行运算，最后的结果的类型和运算中的最大数据类型保持一致

    ​ byte,short,char类型在运算中都是先转换为int类型，再运算，即他们的运算结果一定都是int类型
